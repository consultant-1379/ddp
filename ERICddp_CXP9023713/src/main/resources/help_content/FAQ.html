Frequently Asked Questions about DDC & DDP


1. What is DDP?

DDP (Diagnostic Data Processing/Presentation) comprises the processing engine and front end for presentation of data collected by DDC.
Webinar here, general PPT here for internal discussions and main page for all things DDP is here.

2. What is DDC?

DDC (Diagnostic Data Collection) is a package delivered as part of the OSS-RC, ENIQ, ENM (all variants) , EO-CM, OMBS and EC-SON products. Basic installations are also available for some other products.

3. Why is my site not showing on the main index page?

There are two reasons why a site is not listed on the main index page:
1. A site only listed when once data has been loaded to it.
2 Only active sites are listed by default. This can be changed using the "customize View" option on the top left corner of the DDP index page.

4. How can I get a site removed from DDP?

Sites in DDP are never deleted. Instead they are set "inactive" and data is aged out over time. To get a site changed to inactive, mail PDLDDPCSAD@pdl.internal.ericsson.com or create a TORF Jira on team SMART. To only see active sites, click "Customise View" in the DDP index page and select "Active Only" in the "Site Active " option.

5. How long is the DDP data retained for?

Each DDP server has its own retention limits and these can be seen on the "Check DDP Status" link at the bottom of each DDP page e.g. DDP and DDP2.

9. Why is my file not in the processing queue?

Once a DDC file is loaded, it will be queued for processing. The queue is ordered based on the DDC file date i.e. DDC_Data_<ddmmyy>.tar.gz (not upload date), with the most recent DDC file date taking priority.
If DDC files for the same site on the same day are loaded within 30 minutes of each other, the processing of the second file will be delayed to reduce the load on DDP.

To fix data already uploaded, perform the following to locally stored data in the uploaded folder:
1. Un-compress the file
  gzip -dc /eniq/log/ddc_data/<DDC_server_folder>/DDC_Data_<ddmmyy>.tar.gz | tar xvf - 
2. Create/Update ddp.txt
echo '<sftp_username>' > <ddmmyy>/ddp.txt
Note: This is the SFTP username given when the DDP site was created and there should be no newlines or carriage returns etc. at the end of the string.
3. Tar and Compress the file
 tar cvf - <ddmmyy> | gzip -c > DDC_Data_<ddmmyy>.tar.gz
4. Remove un-compressed folder
rm -r <ddmmyy>
Note: If using OSS version below 11.0, replace DDC_Data_<ddmmyy>.tar.gz with OSS_Data_<ddmmyy>.tar.gz
5. ENM on or after 1.70.26In ENM 1.70.26 or after a script is provided to update ddp.txt. Note: The full path to the DDC file must be provided and the backup file maually deleted afterwards.
Usage: /opt/ericsson/ERICddc/bin/updateDDP <full path to DDC file> <SFTP username>:e.g./opt/ericsson/ERICddc/bin/updateDDP /var/ericsson/ddc_data/ieatlms4926_TOR/uploaded/DDC_Data_190419.tar.gz lmi_ENM423

12. Once a file is sent across to DDP for processing, how long will it be before my data is available?

Assuming all is in order with the data file, then this depends on the traffic and the files from other sites in the queue at the time.
During the normal daily running, outside of maintenance periods, between 30-45 minutes, possibly about an hour in rare circumstances.

19. What is happening when DDP maintenance is running as per the status page?

DDP maintenance executes every night  at 0100.
During maintenance DDC files can still be uploaded, but will not be processed until the maintenance is complete.
In exceptional cases if maintenance is still running at 0600 it is requested to gracefully stop.
Typical maintenance tasks are:
  • Update the help bubbles and FAQ
  • Compress large files
  • Delete temporary files
  • Remap IDs from lookup tables e.g. proc ID when a better match is available for the process string.

25. What can I do about "Problem processing data" errors on the DDP UI

If the error says "Failed to decompress" then on the local laptop in powershell use "Get-FileHash Algorithm MD5 DDC_Data_ddmmyy.tar.gz" and check with "md5sum DDC_Data_ddmmyy.tar.gz" on the ENM deployment.
