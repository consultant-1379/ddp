</DDP_Bubble.hc.nicErrors> <DDP_Bubble.hc.nfsd_getattr>

This check will highlight if the total number of NFS getattr requests per day is beyond expected values. A high number of requests indicates an unexpected use of NFS e.g. rsync usage. It may not necessarily be a fault and each case need to be taken with understanding of the PM file collection use cases for this site. The values are based on internal test systems and are indicative of bad behavior.

Generally, it’s symptomatic of having some application running that performing a full scan of the PM file systems (performing a stat on every file in the filesystem). This can cause the inode cache (currently 2,097,151) on the NAS head to be trashed which in turn causes the file content not to be cached in RAM.

The impact of this is:

  • Increased time to read the PM file systems as all reads have to go to disk.
  • Increase I/O times across the whole system as the SAN is busy servicing the above read requests from the NAS.
  • Server file cache will effectively become empty if the number of files on the NAS is greater that the current (April 2020) inode setting of 2,097,151.

Amber:

  • The number of requests is greater than %requests-warn%.

Red:  

  • The number of requets is greater than %requests%.
